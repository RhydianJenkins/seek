Search API Reference

Overview

The Search API allows you to query your knowledge base using semantic search. It returns the most relevant chunks based on vector similarity.

Endpoint

POST /api/search

Request Body

{
  "query": "How do vector embeddings work?",
  "limit": 5,
  "threshold": 0.7
}

Parameters

Field Type Description
query string The search query text
limit integer Maximum number of results (default: 10)
threshold float Minimum similarity score (0-1)

Response

Returns an array of matching chunks with metadata:

{
  "results": [
    {
      "content": "Vector embeddings are numerical representations...",
      "filename": "intro.md",
      "similarity": 0.89,
      "chunk_id": "abc123"
    }
  ]
}

Example Usage

cURL

curl -X POST http://localhost:8080/api/search \
  -H "Content-Type: application/json" \
  -d '{"query": "RAG implementation", "limit": 3}'

Python

import requests

response = requests.post(
    "http://localhost:8080/api/search",
    json={"query": "RAG implementation", "limit": 3}
)
results = response.json()["results"]

Important Notes

  • Queries are automatically converted to embeddings using the same model used for indexing
  • Similarity scores range from 0 (no match) to 1 (perfect match)
  • Results are sorted by similarity score in descending order
